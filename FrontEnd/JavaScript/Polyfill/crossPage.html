# 跨页面通信

# window.open

# 实现方案

父页面通过 window.open(url, name) 方式打开的子页面可以获取句柄，然后通过 postMessage 完成通讯需求；

# 代码实现

// parent.html
const childPage = window.open('child.html', 'child');

childPage.onload = () => {
	childPage.postMessage('hello', location.origin);
}

// child.html
window.onmessage = e => {
	console.log(e.data);
}

# localStorage

# 实现方案

设置共享区域的 storage，会触发 storage 事件；

# 代码实现

// A.html
localStorage.setItem('message', 'hello');

// B.html
window.onstorage = e => {
  console.log(e.key, e.oldValue, e.newValue);
}

# BroadcastChannel

# 实现方案

和 localStorage 方案基本一致，额外需要初始化；

# 代码实现

// A.html
const channel = new BroadcastChannel('tabs');
channel.postMessage('hello');

// B.html
const channel = new BroadcastChannel('tabs');
channel.onmessage = e => {
	console.log(e.data);
}

# 与 localStorage 的区别

与 localStorage 方案差不多，都是同域、API 简单，BroadcastChannel 方案兼容性差些（chrome > 58），但比localStorage 方案生命周期短（不会持久化），相对干净些；

# SharedWorker

# 实现方案

SharedWorker 本身并不是为了解决通讯需求的，它的设计初衷应该是类似总控，将一些通用逻辑放在 SharedWorker 中处理；

# 代码实现

// A.html
var sharedworker = new SharedWorker('worker.js');
sharedworker.port.start();
sharedworker.port.onmessage = e => {
    console.log(e.data);
}

// B.html
var sharedworker = new SharedWorker('worker.js');
sharedworker.port.start();
sharedworker.port.postMessage('hello');

// worker.js
const ports = []
onconnect = e => {
	const port = e.ports[0];
	ports.push(port);
	port.onmessage = evt => {
		ports.filter(v => {v!== port)
		    .forEach(p => p.postMessage(evt.data));
	}
}
